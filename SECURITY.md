# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public GitHub issue.

Instead, report it privately via the [GitHub Security Advisory](../../security/advisories/new) feature.

Include:
- A clear description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if available)

We aim to respond within 72 hours and will keep you informed throughout the process.

## Scope

This project runs **fully locally** — no data is sent to external servers. 
Network access (if any) is limited to local network resources only.
